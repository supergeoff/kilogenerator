# Naming Conventions

## Files and Directories

- Use `kebab-case` for all file and directory names
- Example: `my-component.ts`, `utils/`, `api-client.js`

## Skill Folders

- Pattern: `skills-{mode-slug}/`
- Example: `skills-code/`, `skills-orchestrator/`, `skills-spec/`

## Workflow Files

- Pattern: `{workflow-name}.md`
- Example: `develop.md`, `generate-specs.md`

## Context Docs

- Pattern: `{topic}.md`
- Example: `naming-conventions.md`, `workflow-patterns.md`

## Skills

- Name in frontmatter must match directory name exactly
- Use English for all skill content
- Description should clearly indicate when to use the skill

## Modes

- Slug: lowercase with hyphens (e.g., `spec`, `code-mode`)
- Names can include emojis for UI display
