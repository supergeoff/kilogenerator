# Documentation Standards

## Language

- All new files must be in **English**
- Existing content in other languages should be migrated when modified

## Frontmatter

All markdown files should include YAML frontmatter:

```yaml
---
title: "Document Title"
description: "Brief description"
---
```

### Required Fields

- `title`: Human-readable document title
- `description`: Brief summary of document purpose

### Optional Fields

- `license`: License name or reference
- `metadata`: Additional key-value pairs

## Format

- Use standard Markdown syntax
- Include headers for major sections
- Use code blocks for examples
- Keep line length reasonable (aim for 80-120 characters)
